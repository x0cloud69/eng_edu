# 1. 개요

본 문서는 Standard Framework 프로젝트에서 Human과 AI Agent 간의 역할과 책임을 명확히 정의한다. 각 Phase의 모든 Activity에 대해 누가 최종 책임을 지고, 누가 실행하며, 누가 보조하는지를 RACI 형식으로 명세한다.

> 💡 이 문서는 Phase 2 Workflow 설계 및 Phase 3 System Prompt 작성의 직접 입력값이다. Agent별 허용 범위(Scope)는 이 문서를 기준으로 정의된다.

# 3. R&R 핵심 원칙

## 3.1 Human이 반드시 담당하는 것 (절대 Agent에게 위임 불가)

| 항목                    | 이유                               | 위반 시 처리              |
|-----------------------|----------------------------------|----------------------|
| 최종 승인 (H-A)           | 책임(Accountability)은 항상 인간 조직에 귀속 | 배포 차단 + 재승인 필요       |
| Authority Level 선정    | AI 자율성 수준은 Human이 결정             | Governance Board 재검토 |
| Governance Charter 서명 | 법적/조직적 책임 귀속                     | 문서 무효 처리             |
| 예외 승인                 | 표준 위반에 대한 책임 귀속                  | 예외 기록 + 만료 기간 설정     |
| Risk 최종 판단 (Critical) | 생명/안전/법적 책임 직접 영향                | 즉시 에스컬레이션            |

## 3.2 Agent가 자율적으로 할 수 있는 것 (L1 기준)

| Agent          | 자율 실행 가능                   | Human 승인 필요                  | 절대 불가                |
|----------------|----------------------------|------------------------------|----------------------|
| Orchestrator   | Workflow 조정, Escalation 판단 | Decision Gate 통과, Context 변경 | 전략 해석, 자동 권한 상승      |
| UIUX Agent     | Persona 분석 보조, 시나리오 초안     | 최종 Persona 확정                | Context 직접 수정        |
| Frontend Agent | 컴포넌트 코드 생성                 | 최종 배포 코드 확정                  | any 타입 사용, API 직접 호출 |
| Backend Agent  | 코드 생성, 쿼리 작성               | DB 변경, 외부 API 호출             | Audit Log 없는 실행      |
| Test Agent     | 시나리오 자동 실행, KPI 측정         | Evaluation Report 확정         | 승인 없이 재배포 요청         |
