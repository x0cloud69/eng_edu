# Markdown 명령어 정리 (Obsidian 기준)

이 문서는 Obsidian에서 자주 사용하는 Markdown 명령어를 정리한
가이드입니다. 특히 \`\`\` 와 \`\`\`\` 같은 백틱(backtick) 사용 방법을
포함합니다.

------------------------------------------------------------------------

# 1. 제목 (Headings)

    # H1 제목
    ## H2 제목
    ### H3 제목
    #### H4 제목

------------------------------------------------------------------------

# 2. 강조 (Bold / Italic)

    **굵은 글씨**
    *기울임*
    ~~취소선~~

------------------------------------------------------------------------
# 3. 리스트
## 일반 리스트

    - 항목 1
    - 항목 2

## 번호 리스트

    1. 첫번째
    2. 두번째

## 체크박스

    - [ ] 해야 할 일
    - [x] 완료

------------------------------------------------------------------------

# 4. 링크

## Obsidian 내부 링크

    [[Persona]]
    [[Architecture]]

## 외부 링크

    [OpenAI](https://openai.com)

------------------------------------------------------------------------

# 5. 코드 (Backtick 사용법)

## 1) 인라인 코드

    `code`

예시: `Agent Role`

------------------------------------------------------------------------

## 2) 코드 블록 (3개 백틱)

    ```
    Agent: Planner
    Role: QA
    ```

설명: - \`\`\` 는 코드 블록 시작/종료 - 개발 명세, 프롬프트, 구조 작성
시 사용

------------------------------------------------------------------------

## 3) 4개 백틱 (\`\`\`\`) 사용 이유

코드 안에 \`\`\` 가 포함될 때 사용합니다.

예시:

    ```
    여기 안에 ``` 가 있어도 깨지지 않음
    ```

즉, - \`\`\` = 일반 코드 블록 - \`\`\`\` = 코드 블록 안에 코드 블록을
표현할 때

------------------------------------------------------------------------

# 정리

백틱(backtick) 사용 규칙:

-   \` = 인라인 코드
-   \`\`\` = 코드 블록
-   \``= 코드 안에` 표시할 때

Obsidian에서 명령어처럼 보이는 대부분의 문법은 실제로는 Markdown 구조
표현입니다.
