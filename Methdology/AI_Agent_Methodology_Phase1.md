# AI Agent 방법론

# Phase 1 -- Vision & Role 정의 단계

------------------------------------------------------------------------

## 1. Phase 개요

### 1.1 이 단계의 목적

본 단계는 AI Agent 시스템의 전략적 방향성과 거버넌스 기준을 수립하는
단계이다.

목적: - Agent 존재 이유 정의 - 해결하고자 하는 문제의 명확화 -
Human--Agent 책임 경계 설정 - 이후 단계의 설계 기준 확립

본 단계는 전체 설계의 Design Authority Layer 역할을 수행한다.

------------------------------------------------------------------------

## 2. Phase 목표

Phase 1 종료 시 다음 항목이 반드시 정의 및 승인되어야 한다:

1.  Problem Definition
2.  Persona 정의
3.  Agent 역할 정의
4.  권한 범위(Authority Boundary) 정의
5.  Human-in-the-loop 구조 정의
6.  거버넌스 및 윤리 기준 정의

본 단계 승인 이전에는 기술 구현을 진행하지 않는다.

------------------------------------------------------------------------

## 3. 세부 정의 항목

### 3.1 Problem Definition

정의해야 할 항목: - 핵심 문제 정의 (누가, 무엇 때문에, 어떤 영향을
받는가) - 문제의 영향 분석 - 기존 해결 방식의 한계 - AI Agent 접근이
필요한 이유

산출물: - Problem Definition 문서

------------------------------------------------------------------------

### 3.2 Persona 정의

정의해야 할 항목: - 주요 타겟 사용자 - 행동 특성 - 사용 맥락 - 감정적 /
기능적 요구 - 리스크 특성 (필요 시)

산출물: - Persona Specification 문서 - Usage Scenario 문서

------------------------------------------------------------------------

### 3.3 Agent 역할 정의

정의해야 할 항목: - Agent 유형 (보조 / 추천 / 의사결정 / 자율 실행) -
수행 범위 - 명확한 Non-goal (하지 않는 것) - 성공 기준 - 실패 정의

산출물: - Agent Role Definition 문서

------------------------------------------------------------------------

### 3.4 권한(Authority) 모델 정의

정의해야 할 항목: - 자율성 수준 (L0--L3) - 승인 필요 여부 - Override
규칙 - Escalation 기준

예시: - L0: 정보 제공 - L1: 추천 - L2: 자동 실행 (승인 필요) - L3: 완전
자율 실행

산출물: - Authority Boundary 정의 문서

------------------------------------------------------------------------

### 3.5 Human-in-the-loop 구조

정의해야 할 항목: - Human 개입 조건 - 승인 프로세스 - 모니터링 책임자 -
최종 책임 주체

산출물: - Human--AI Interaction 모델 문서

------------------------------------------------------------------------

### 3.6 거버넌스 및 윤리 기준

정의해야 할 항목: - 법적 고려 사항 - 데이터 민감도 분류 - 윤리적 한계 -
투명성 정책 - 사고 발생 시 책임 구조

산출물: - Governance Charter

------------------------------------------------------------------------

## 4. Role & Responsibility

### 4.1 Human 책임

-   비즈니스 목표 승인
-   리스크 수용 여부 결정
-   권한 범위 승인
-   최종 의사결정

### 4.2 Agent 책임

-   분석 초안 생성
-   시나리오 시뮬레이션
-   추천 생성
-   위험 점수 산정 (해당 시)

------------------------------------------------------------------------

## 5. Work Breakdown Structure (WBS)

### 5.1 Problem Framing

-   5.1.1 Problem Statement 작성
-   5.1.2 이해관계자 목표 정렬
-   5.1.3 시장 / 상황 분석

### 5.2 Persona 정의

-   5.2.1 핵심 Persona 선정
-   5.2.2 행동 패턴 정의
-   5.2.3 사용자 시나리오 작성
-   5.2.4 사용 맥락 요구사항 정의

### 5.3 Agent Role Mapping

-   5.3.1 Role Matrix 작성
-   5.3.2 Authority Boundary 정의
-   5.3.3 Non-goal 문서화
-   5.3.4 Escalation 원칙 정의

------------------------------------------------------------------------

## 7. 완료 기준 (Exit Criteria)

다음 조건을 모두 충족해야 Phase 1이 완료된 것으로 판단한다:

-   Agent를 한 문장으로 정의 가능하다.
-   권한 경계가 명확하다.
-   Human Override 정책이 문서화되어 있다.
-   Non-goal이 명확히 정의되어 있다.
-   거버넌스 책임 주체가 지정되어 있다.
