## 1. 문서 개요

### 1.1 목적

본 문서는 Python FastAPI(Backend) + Next.js(Frontend) 기반 Boilerplate 프로젝트에 공통 기반 파일을 추가할 때의 올바른 **작업 순서**, 각 파일의 **역할**, 그리고 **구체적인 구현 방향**을 정리한 가이드입니다.

공통 기반 파일은 프로젝트 전체에서 재사용되는 핵심 코드이므로, 의존성 순서를 지키지 않으면 빌드 오류나 런타임 오류가 발생할 수 있습니다.

### 1.2 기술 스택

| 구분 | 기술 |
|------|------|
| Backend | Python 3.x · FastAPI · SQLAlchemy 2.0 (async) · Pydantic |
| Frontend | TypeScript · React · Next.js (App Router) |
| DB | PostgreSQL 16 |
| Cache | Redis 7 |
| 인프라 | Docker Compose (로컬 개발) |

### 1.3 전체 작업 흐름

```
Phase 1. 인프라 / 루트 설정
     ↓
Phase 2. Backend 공통 기반 (FastAPI)
     ↓
Phase 3. Frontend 공통 기반 (Next.js)
     ↓
Phase 4. 통합 연결 및 검증
```

> 각 Phase는 이전 Phase가 완료된 후 시작하는 것을 원칙으로 합니다.  
> Phase 내부에서는 의존성 순서(낮은 것 → 높은 것)를 반드시 지켜야 합니다.

### 2.2 파일별 상세 설명

#### 2.2.1 docker-compose.yml

PostgreSQL, Redis, Backend(FastAPI), Frontend(Next.js) 4개 컨테이너를 정의합니다.

```yaml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - '5432:5432'

  redis:
    image: redis:7-alpine
    ports:
      - '6379:6379'

  backend:
    build: ./backend
    env_file: .env
    depends_on: [db, redis]
    ports:
      - '8000:8000'

  frontend:
    build: ./frontend
    env_file: .env
    depends_on: [backend]
    ports:
      - '3000:3000'

volumes:
  postgres_data:
```

#### 2.2.2 .env.example

실제 `.env` 파일은 Git에 올리지 않으므로, 팀원들이 필요한 환경변수를 파악할 수 있도록 반드시 작성합니다.

```bash
# Database
POSTGRES_DB=myapp
POSTGRES_USER=postgres
POSTGRES_PASSWORD=changeme
DATABASE_URL=postgresql+asyncpg://postgres:changeme@db:5432/myapp

# Redis
REDIS_URL=redis://redis:6379/0

# Application
SECRET_KEY=your-secret-key-here
DEBUG=true
CORS_ORIGINS=http://localhost:3000

# Frontend
NEXT_PUBLIC_API_URL=http://localhost:8000
```

#### 2.2.3 Makefile

자주 사용하는 Docker Compose 명령어를 단축키로 등록하여 개발 생산성을 높입니다.

```makefile

	docker-compose up -d

	docker-compose down

	docker-compose logs -f

migrate:
	docker-compose exec backend alembic upgrade head

shell-backend:
	docker-compose exec backend bash
```

```python
from functools import lru_cache

class Settings(BaseSettings):

@lru_cache()
```

#### Step 5 — 공통 베이스 모델 (`models/base.py`)

모든 DB 모델이 공통으로 상속할 베이스 클래스입니다.

```python
from sqlalchemy import DateTime, Integer
from sqlalchemy.orm import Mapped, mapped_column
from datetime import datetime, timezone
from app.db.base import Base

class TimestampMixin:
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        default=lambda: datetime.now(timezone.utc)
    )
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        default=lambda: datetime.now(timezone.utc),
        onupdate=lambda: datetime.now(timezone.utc)
    )

class BaseModel(Base, TimestampMixin):
    __abstract__ = True
    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
```

#### Step 6 — 공통 Pydantic 스키마 (`schemas/base.py`)

모든 API 응답 형식을 통일합니다.

```python
from pydantic import BaseModel
from typing import Generic, TypeVar, Optional

T = TypeVar('T')

class ApiResponse(BaseModel, Generic[T]):
    success: bool = True
    data: Optional[T] = None
    message: str = ''

class PaginatedResponse(BaseModel, Generic[T]):
    items: list[T]
    total: int
    page: int
    size: int
    total_pages: int
```

#### Step 7 — Redis 캐시 래퍼 (`core/cache.py`)

```python
import redis.asyncio as redis
import json
from app.core.config import settings

redis_client: redis.Redis = None

async def get_redis() -> redis.Redis:
    global redis_client
    if redis_client is None:
        redis_client = redis.from_url(settings.redis_url)
    return redis_client

async def cache_get(key: str):
    r = await get_redis()
    value = await r.get(key)
    return json.loads(value) if value else None

async def cache_set(key: str, value, ttl: int = 300):
    r = await get_redis()
    await r.setex(key, ttl, json.dumps(value))

async def cache_delete(key: str):
    r = await get_redis()
    await r.delete(key)
```

#### Step 10 — FastAPI 앱 진입점 (`main.py`)

모든 파일이 준비된 후 **마지막**에 작성합니다.

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from app.core.config import settings
from app.core.logging import setup_logging
from app.db.session import engine

@asynccontextmanager
async def lifespan(app: FastAPI):
    setup_logging()
    yield
    await engine.dispose()

app = FastAPI(lifespan=lifespan, title='My API', debug=settings.debug)

app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins,
    allow_credentials=True,
    allow_methods=['*'],
    allow_headers=['*'],
)

# 라우터 등록 (도메인 라우터 추가 시 여기에 include_router 추가)
```

```typescript
import axios from 'axios';

const apiClient = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
  timeout: 10000,
  headers: { 'Content-Type': 'application/json' },
});

// 요청 인터셉터 — 토큰 자동 첨부
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// 응답 인터셉터 — 에러 공통 처리
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('access_token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default apiClient;
```

#### Step 6 — 공통 훅 (`hooks/usePagination.ts`)

```typescript
import { useState } from 'react';

export function usePagination(initialPage = 1, initialSize = 20) {
  const [page, setPage] = useState(initialPage);
  const [size, setSize] = useState(initialSize);

  const goToPage = (newPage: number) => setPage(Math.max(1, newPage));
  const reset = () => setPage(1);

  return { page, size, setPage: goToPage, setSize, reset };
}
```

## 7. 참고사항 및 주의점

### 7.1 의존성 순서를 반드시 지켜야 하는 이유

Python에서 모듈을 `import`할 때 해당 모듈이 의존하는 다른 모듈이 먼저 존재해야 합니다. 예를 들어 `session.py`는 `config.py`의 `settings`를 import하므로, `config.py`가 먼저 완성되어 있어야 합니다. 순서를 무시하면 `ImportError` 또는 순환 참조(Circular Import) 오류가 발생합니다.

### 7.2 환경변수 관리 주의사항

- `.env` 파일은 절대 Git에 커밋하지 않습니다. `.gitignore`에 반드시 추가합니다.
- `.env.example` 파일은 실제 값 없이 키만 포함하여 Git에 커밋합니다.
- 새 환경변수 추가 시 `.env.example`도 동시에 업데이트합니다.
- Backend의 Pydantic BaseSettings와 Frontend의 `.env.local` 네이밍을 일치시켜 혼란을 줄입니다.

### 7.3 SQLAlchemy async 사용 시 주의사항

- `asyncpg` 드라이버를 사용해야 합니다. `DATABASE_URL`에 `postgresql+asyncpg://` 형식으로 작성합니다.
- 모든 DB 쿼리 함수는 `async def`로 선언하고 `await`를 사용해야 합니다.
- 세션을 사용하는 곳에서는 반드시 `async with` 또는 `Depends(get_db)`를 통해 세션을 주입받습니다.

### 7.4 Redis 연결 관리

- `redis-py`의 async 클라이언트(`redis.asyncio`)를 사용합니다.
- Redis 연결은 앱 시작 시 한 번만 생성하고(싱글톤 패턴) 재사용합니다.
- 캐시 키 네이밍은 `namespace:entity:id` 형태(예: `user:detail:123`)로 통일합니다.

### 7.5 Frontend React Query 사용 가이드라인

- 서버 데이터(API 응답)는 항상 React Query로 관리합니다. 클라이언트 UI 상태(모달 열림 여부 등)는 `useState`로 관리합니다.
- `queryKey`는 배열 형식으로 `['users', userId]`처럼 계층적으로 구성합니다.
- Optimistic Update가 필요한 뮤테이션은 `onMutate`, `onError`, `onSettled` 콜백을 활용합니다.
