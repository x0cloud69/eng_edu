## 1. Boilerplate란?

`backend/src/core/` 안에 있는 **공통 기반 파일들**이다.
모든 Phase 3 코드는 이 파일들을 상속·재사용하므로, 수정 시 영향 범위가 넓다.

| 파일 | 역할 |
|------|------|
| `base_agent.py` | 모든 Agent의 부모 클래스 |
| `base_model.py` | 모든 DB 모델의 부모 클래스 |
| `llm_provider.py` | LLM 호출 추상화 인터페이스 |
| `security.py` | JWT 인증, RBAC, 공통 API 응답 |
| `database.py` | SQLAlchemy 2.0 async DB 연결 |

---

## 2. 수정 전 반드시 확인할 것

```
1. 어떤 파일을 수정하는가?
2. 그 파일을 상속/사용하는 하위 파일은 무엇인가?
3. DB 스키마 변경이 포함되는가? → alembic 필요
4. phase3_context.json 업데이트가 필요한가?
```

## 3. 파일별 수정 절차

### 3-1. base_agent.py 수정 (Agent 로직 변경)

**영향 범위:** BaseAgent를 상속한 모든 Agent 파일

```bash
# Step 1. 파일 수정 (Cursor 사용)
backend/src/core/base_agent.py

# Step 2. 영향받는 Agent 파일 확인
grep -r "BaseAgent" backend/src/

# Step 3. Docker 재시작
docker compose -f docker-compose.dev.yml restart backend

# Step 4. 테스트
POST /api/v1/agents/invoke 테스트
```

---

### 3-2. base_model.py 수정 (DB 모델 변경)

**영향 범위:** TenantBaseModel을 상속한 모든 DB 모델

```bash
# Step 1. 파일 수정 (Cursor 사용)
backend/src/core/base_model.py

# Step 2. 영향받는 모델 확인
grep -r "TenantBaseModel" backend/src/

# Step 3. 마이그레이션 생성 (컨테이너 안에서 실행)
docker compose -f docker-compose.dev.yml exec backend \
  alembic revision --autogenerate -m "변경내용 설명"

# Step 4. DB에 반영
docker compose -f docker-compose.dev.yml exec backend \
  alembic upgrade head

# Step 5. Docker 재시작
docker compose -f docker-compose.dev.yml restart backend
```

### 3-3. security.py 수정 (인증/응답 형식 변경)

**영향 범위:** 모든 API 엔드포인트 (api_success, api_error 사용하는 곳)

backend/src/core/security.py

## 4. phase3_context.json 업데이트 기준

Boilerplate 수정 후 아래 항목이 바뀌었으면 **반드시** `phase3_context.json`도 업데이트한다.

| 변경 내용 | 업데이트할 섹션 |
|----------|---------------|
| 새 클래스/함수 추가 | `boilerplate[].key_classes` 또는 `key_functions` |
| do/dont 규칙 변경 | `boilerplate[].do` / `boilerplate[].dont` |
| 코딩 규칙 변경 | `code_generation_rules` |
| Agent 템플릿 변경 | `agent_template` |
| 모듈 폴더 구조 변경 | `module_template` |
| 새 API 엔드포인트 추가 | `api_endpoints.existing` |

### 업데이트 후 Cursor에 알리는 방법
```
@phase3_context.json

phase3_context.json이 업데이트 되었어.
변경된 내용: (변경사항 설명)
앞으로 코드 생성 시 새 규칙을 반영해줘.
```

## 5. 신규 Boilerplate 파일 추가 시

완전히 새로운 공통 파일이 필요할 때 절차:

```
Step 1. backend/src/core/ 에 파일 생성 (Cursor 사용)
Step 2. phase3_context.json boilerplate 섹션에 항목 추가
        - path, purpose, rule, key_classes, do, dont 작성
Step 3. code_generation_rules 에 관련 규칙 추가
Step 4. Docker 재시작
Step 5. 테스트
```

## 6. 자주 쓰는 명령어 모음

```bash
# 컨테이너 상태 확인
docker compose -f docker-compose.dev.yml ps

# Backend 재시작
docker compose -f docker-compose.dev.yml restart backend

# 마이그레이션 생성 (컨테이너 안에서)
docker compose -f docker-compose.dev.yml exec backend \
  alembic revision --autogenerate -m "설명"

# 마이그레이션 적용
docker compose -f docker-compose.dev.yml exec backend \
  alembic upgrade head

# 마이그레이션 되돌리기
docker compose -f docker-compose.dev.yml exec backend \
  alembic downgrade -1

# DB 테이블 목록 확인
docker compose -f docker-compose.dev.yml exec db \
  psql -U postgres -d my_framework_dev -c "\dt"

# Backend 로그 확인
docker compose -f docker-compose.dev.yml logs -f backend
```

## 7. 체크리스트 (수정 완료 후)

```
□ 수정한 파일 저장 확인
□ 영향받는 하위 파일 확인 및 수정
□ DB 변경이 있으면 alembic 마이그레이션 완료
□ docker restart backend 완료
□ phase3_context.json 업데이트 완료
□ Cursor에 변경사항 알림
□ 관련 API 테스트 완료
□ 이 문서 업데이트 (필요시)
```
