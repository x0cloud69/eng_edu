# 1. 개요

Tool은 Agent가 외부 시스템(DB, API, 파일 시스템, 알림 서비스)에 접근하는 **최하위 인터페이스**다. Agent는 Tool을 직접 호출하지 않고 반드시 **Skill → Tool** 순서로 경유한다.

> 💡 Tool은 "무엇에 접근하는가"를 정의하고, Skill은 "어떻게 사용하는가"를 정의한다. Tool Config는 접근 권한, 연결 설정, Error Handling의 단일 진실 출처(Single Source of Truth)다.

## 1.1 Tool 분류 체계

```
Tool 분류
├── Internal Tools    — 프로젝트 내부 시스템 (DB, Redis, 내부 API)
├── Agent Tools       — LLM 호출, Agent 간 통신
└── External Tools    — 외부 서비스 (알림, 이메일, Webhook)
```

## 1.2 Tool 공통 인터페이스

```python
class ToolBase:
    tool_id: str
    tool_name: str
    version: str
    allowed_agents: list[str]
    requires_approval: bool
    timeout_seconds: int = 30
    retry_count: int = 3
    retry_delay_seconds: float = 1.0

    async def call(self, input: ToolInput) -> ToolOutput:
        raise NotImplementedError

    async def health_check(self) -> bool:
        raise NotImplementedError
```

## 1.3 Tool 호출 흐름

```
Agent
  ↓ Skill 경유
Skill Registry
  ↓ 권한 확인 + 승인 체크
Tool Registry
  ↓ Health Check
Tool
  ↓ 실행
  ├── 성공 → 결과 반환 + audit_log
  └── 실패 → Retry → 초과 시 Escalate + audit_log(failure)
```

## 2.1 db_query — PostgreSQL 쿼리 실행

| 항목 | 값 |
|------|---|
| Tool ID | `internal.db_query` |
| 버전 | v1.0.0 |
| 허용 Agent | Back-A, Orch, Evol-A (읽기), Test-A (테스트 DB 전용) |
| 승인 필요 | 쓰기(INSERT/UPDATE/DELETE): **P1 필수** / 읽기: 불필요 |
| Timeout | 30초 |
| Retry | 3회 (지수 백오프) |

**연결 설정:**
```python
# settings.py 경유 — 코드 내 직접 연결 정보 금지
DB_CONFIG = {
    "host": settings.POSTGRES_HOST,
    "port": settings.POSTGRES_PORT,
    "database": settings.POSTGRES_DB,
    "user": settings.POSTGRES_USER,
    "password": settings.POSTGRES_PASSWORD,
    "pool_size": 20,
    "max_overflow": 10,
    "pool_timeout": 30,
    "pool_recycle": 1800,
    "echo": settings.DB_ECHO,         # 프로덕션: False
    "echo_pool": settings.DB_ECHO,
}
```

**입력 스키마:**
```python
class DbQueryInput(BaseModel):
    trace_id: UUID
    agent_id: str
    operation: Literal["SELECT", "INSERT", "UPDATE", "DELETE"]
    query: str            # SQLAlchemy ORM 쿼리 (raw SQL 금지)
    params: dict = {}
    use_transaction: bool = True

    @validator("operation")
    def write_requires_approval(cls, v, values):
        if v in ["INSERT", "UPDATE", "DELETE"]:
            # 호출 Skill에서 P1 승인 확인 필수
            pass
        return v
```

**출력 스키마:**
```python
class DbQueryOutput(BaseModel):
    trace_id: UUID
    rows_affected: int
    result: list[dict] | None
    execution_time_ms: float
    transaction_id: str | None
```

**Error Handling:**

| 오류 유형 | 처리 방법 |
|---------|---------|
| Connection Error | 3회 Retry (1s → 2s → 4s) → Escalate to Arch |
| Query Timeout (30s) | 즉시 롤백 + Escalate |
| Integrity Error (FK, Unique) | 롤백 + 상세 오류 반환 (재시도 없음) |
| Permission Error | 즉시 차단 + 보안 알림 |

**연결 설정:**
```python
REDIS_CONFIG = {
    "host": settings.REDIS_HOST,
    "port": settings.REDIS_PORT,
    "password": settings.REDIS_PASSWORD,
    "db": 0,
    "decode_responses": True,
    "socket_timeout": 5,
    "socket_connect_timeout": 5,
    "retry_on_timeout": True,
    "health_check_interval": 30,
}
```

**입력 스키마:**
```python
class RedisCacheInput(BaseModel):
    trace_id: UUID
    operation: Literal["GET", "SET", "DELETE", "EXISTS", "EXPIRE", "INCR"]
    key: str
    value: str | None = None
    ttl_seconds: int | None = None   # SET 시 TTL 필수 권장

    @validator("key")
    def validate_key_format(cls, v):
        # 키 형식: "{prefix}:{identifier}" 필수
        if ":" not in v:
            raise ValueError("Redis key must follow '{prefix}:{id}' format")
        return v
```

**키 네임스페이스 규칙:**

| 접두사 | 용도 | TTL 기준 |
|-------|------|---------|
| `session:{user_id}` | 사용자 세션 | 24h |
| `refresh_token:{user_id}:{jti}` | Refresh Token | 30일 |
| `login_attempts:{email}` | 로그인 실패 횟수 | 15분 |
| `approval:{approval_id}` | 승인 요청 캐시 | 48h |
| `agent_run:{trace_id}` | Agent 실행 상태 | 1h |
| `kpi:{metric_name}:{date}` | KPI 메트릭 | 7일 |
| `rate_limit:{user_id}:{endpoint}` | API Rate Limit | 1분 |

## 5.2 권한 위반 처리

모든 권한 위반은 아래 순서로 처리된다.

```
① Tool Registry가 권한 매트릭스 조회
② 허용되지 않은 Agent 감지 시 → 호출 즉시 차단
③ PermissionViolationError 발생
④ audit_logs에 PERMISSION_VIOLATION 기록
⑤ Orch에게 보안 이벤트 보고
⑥ 해당 Agent 세션 일시 중단 (Orch 판단)
```

# 6. Error Handling 전략

## 6.1 오류 분류 및 처리 기준

| 오류 코드 | 분류 | Retry | Escalate | 감사 로그 |
|---------|------|:-----:|:--------:|:-------:|
| `TOOL_TIMEOUT` | 일시적 | ✅ 3회 | 3회 초과 시 | ✅ |
| `TOOL_RATE_LIMIT` | 일시적 | ✅ 대기 후 | - | ✅ |
| `TOOL_CONNECTION_ERROR` | 인프라 | ✅ 3회 | 3회 초과 시 | ✅ |
| `TOOL_PERMISSION_ERROR` | 보안 | ❌ | 즉시 | ✅ |
| `TOOL_VALIDATION_ERROR` | 입력 오류 | ❌ | - | ✅ |
| `TOOL_NOT_FOUND` | 설정 오류 | ❌ | 즉시 | ✅ |
| `APPROVAL_REQUIRED` | 승인 필요 | ❌ | 승인 요청 | ✅ |

## 6.2 Circuit Breaker 설정

```python
CIRCUIT_BREAKER_CONFIG = {
    "db_query": {
        "failure_threshold": 5,   # 5회 연속 실패 시 차단
        "recovery_timeout": 60,   # 60초 후 복구 시도
    },
    "llm_call": {
        "failure_threshold": 3,
        "recovery_timeout": 30,
    },
    "external_webhook": {
        "failure_threshold": 3,
        "recovery_timeout": 120,
    }
}
```

## 6.3 Retry 전략

```python
# 지수 백오프 + Jitter
def calculate_retry_delay(attempt: int, base: float = 1.0) -> float:
    delay = base * (2 ** attempt)           # 1s → 2s → 4s
    jitter = random.uniform(0, delay * 0.1) # 10% Jitter
    return min(delay + jitter, 30.0)        # 최대 30초
```
