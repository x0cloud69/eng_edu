# 1. 개요

Memory는 AI Agent가 과거 실행 이력, 사용자 컨텍스트, 도메인 지식을 유지하고 재활용하는 구조다. 적절한 Memory 설계 없이는 Agent가 대화 맥락을 잃거나, 동일 실수를 반복하거나, 불필요한 LLM 호출이 증가한다.

> 💡 Memory는 Agent의 "작업 기억(Working Memory)"과 "장기 기억(Long-term Memory)"을 모두 포함한다. 공공기관 적용 시스템 특성상 모든 Memory는 개인정보보호법 및 데이터 보존 정책의 적용을 받는다.

## 1.1 Memory 4계층 구조

```
Layer 1. Context Window Memory  — 현재 대화 세션 내 임시 메모리 (LLM 토큰 컨텍스트)
Layer 2. Session Memory         — 세션 단위 작업 상태 (Redis, TTL: 24h)
Layer 3. Episodic Memory        — Agent 실행 에피소드 로그 (PostgreSQL, trajectory_logs)
Layer 4. Semantic Memory        — 도메인 지식·패턴·임베딩 (PostgreSQL + pgvector)
```

## 1.2 Memory 유형별 특성 비교

| 구분 | Layer 1 | Layer 2 | Layer 3 | Layer 4 |
|------|---------|---------|---------|---------|
| 저장소 | LLM 컨텍스트 | Redis | PostgreSQL | PostgreSQL + pgvector |
| 범위 | 단일 대화 | 단일 세션 | 영구 | 영구 |
| TTL | 세션 종료 시 소멸 | 24h | 보존 정책 기준 | 버전 관리 |
| 조회 방식 | 자동 (LLM) | 키 조회 | SQL 쿼리 | 벡터 유사도 |
| 개인정보 포함 | 가능 | 가능 | 가능 | 익명화 |
| 승인 필요 | 불필요 | 불필요 | 읽기: 불필요 / 삭제: P1 | 수정: P1 |

# 2. Layer 1 — Context Window Memory

## 2.1 정의

LLM API 호출 시 `messages` 배열에 포함되는 대화 이력이다. Agent가 현재 작업의 맥락을 유지하는 데 사용한다.

## 2.2 관리 규칙

```python
class ContextWindowManager:
    MAX_TOKENS = 100_000        # 최대 컨텍스트 크기
    RESERVE_TOKENS = 8_192      # 출력용 예약 토큰
    WORKING_TOKENS = 91_808     # 실제 사용 가능 토큰

    def build_messages(
        self,
        system_prompt: str,
        history: list[dict],
        new_message: str,
    ) -> list[dict]:
        """토큰 초과 시 오래된 메시지부터 제거 (sliding window)"""
        messages = history.copy()
        messages.append({"role": "user", "content": new_message})

        while self._count_tokens(messages) > self.WORKING_TOKENS:
            # 가장 오래된 user-assistant 쌍 제거 (system 메시지 유지)
            if len(messages) > 2:
                messages.pop(1)
                messages.pop(1)
            else:
                break

        return messages
```

## 2.3 민감정보 필터링

```python
SENSITIVE_PATTERNS = [
    r"\b\d{13}\b",                     # 주민등록번호 (13자리)
    r"\b\d{4}-\d{4}-\d{4}-\d{4}\b",  # 신용카드 번호
    r"(?i)(password|secret|api_key)\s*[:=]\s*\S+",  # 비밀번호/키
]

def sanitize_context(content: str) -> str:
    """컨텍스트 저장 전 민감정보 마스킹"""
    for pattern in SENSITIVE_PATTERNS:
        content = re.sub(pattern, "[REDACTED]", content)
    return content
```

# 3. Layer 2 — Session Memory (Redis)

## 3.1 저장 구조

| Redis 키 | 저장 데이터 | TTL | 접근 Agent |
|---------|----------|-----|---------|
| `agent_run:{trace_id}` | 현재 실행 상태 스냅샷 | 1h | Orch, 해당 Agent |
| `agent_context:{agent_id}:{session_id}` | Agent 세션 컨텍스트 | 24h | 해당 Agent |
| `approval_state:{approval_id}` | 승인 요청 대기 상태 | 48h | Orch |
| `workflow_state:{workflow_id}` | Workflow 진행 상태 | 24h | Orch |
| `tool_call_cache:{hash}` | 동일 Tool 호출 캐시 | 5m | 해당 Agent |

## 3.2 Agent 실행 상태 스키마

```python
class AgentRunState(BaseModel):
    """agent_run:{trace_id} 에 저장되는 구조"""
    trace_id: UUID

    status: Literal["initializing", "running", "waiting_approval",
                    "escalated", "completed", "failed"]
    current_step: str
    steps_completed: list[str]
    context_summary: str          # LLM 요약 (토큰 절약)
    pending_approval_id: UUID | None
```

## 3.3 Session Memory 갱신 규칙

```
Agent 실행 단계마다:
  1. Redis에서 현재 상태 조회 (GET agent_run:{trace_id})
  2. 작업 수행
  3. 상태 갱신 후 저장 (SET agent_run:{trace_id} + EXPIRE 3600)
  4. audit_log에 상태 변경 기록

세션 종료 시:
  1. 최종 상태를 PostgreSQL trajectory_logs에 영구 저장
  2. Redis 키 삭제 (DEL agent_run:{trace_id})
```

## 4.1 trajectory_logs 테이블 스키마

## 4.2 Episodic Memory 조회 규칙

```python
class EpisodicMemoryRetriever:

    async def get_recent_failures(
        self,
        agent_id: str,
        limit: int = 10,
        days: int = 7,
    ) -> list[TrajectoryLog]:
        """최근 실패 에피소드 조회 — Evol-A 분석에 사용"""
        return await self.repo.query(
            agent_id=agent_id,
            status="failure",
            from_date=datetime.now() - timedelta(days=days),
            limit=limit,
            order_by="started_at DESC",
        )

    async def get_similar_episodes(
        self,
        input_summary: str,
        agent_id: str,
        limit: int = 5,
    ) -> list[TrajectoryLog]:
        """유사한 과거 에피소드 조회 — Layer 4 벡터 검색 활용"""
        embedding = await self.embedding_service.encode(input_summary)
        return await self.repo.vector_search(
            embedding=embedding,
            agent_id=agent_id,
            limit=limit,
            min_similarity=0.85,
        )
```

# 5. Layer 4 — Semantic Memory (pgvector)

## 5.1 저장 구조

Semantic Memory는 Agent가 도메인 지식, 패턴, 개선 이력을 장기적으로 학습하고 재사용하기 위한 벡터 저장소다.

```sql
-- pgvector 확장 필요
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE semantic_memories (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id        VARCHAR(50),          -- NULL이면 전체 공유 지식
    memory_type     VARCHAR(50) NOT NULL, -- "knowledge|pattern|improvement|constraint"
    content         TEXT NOT NULL,        -- 저장할 지식 내용 (익명화)
    embedding       vector(1536),         -- OpenAI text-embedding-3-small 기준
    source_trace_id UUID,                 -- 학습 출처 trajectory_log
    tags            TEXT[] DEFAULT '{}',
    confidence      DECIMAL(3,2),         -- 0.00 ~ 1.00
    use_count       INTEGER DEFAULT 0,
    is_active       BOOLEAN DEFAULT TRUE,
    version         INTEGER DEFAULT 1,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_semantic_embedding ON semantic_memories
    USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
CREATE INDEX idx_semantic_agent_type ON semantic_memories(agent_id, memory_type);
CREATE INDEX idx_semantic_tags ON semantic_memories USING GIN(tags);
```

## 5.2 Memory 유형별 정의

| memory_type | 설명 | 학습 주체 | 적용 대상 |
|------------|------|---------|---------|
| `knowledge` | 도메인 규칙·정책·제약사항 | 사전 등록 (Human) | 전체 Agent |
| `pattern` | 반복되는 성공 실행 패턴 | Evol-A (자동) | 해당 Agent |
| `improvement` | 개선 제안 및 적용 이력 | Evol-A (승인 후) | 해당 Agent |
| `constraint` | 금지 패턴·실패 원인 | Evol-A + Test-A | 전체 Agent |

## 5.3 Semantic Memory Retrieval Rule

```python
class SemanticMemoryRetriever:

    RETRIEVAL_CONFIG = {
        "max_results": 5,
        "min_similarity": 0.80,     # 80% 이상 유사도만 반환
        "max_tokens_budget": 2000,  # 컨텍스트 주입 시 최대 토큰
    }

    async def retrieve(
        self,
        query: str,
        agent_id: str,
        memory_types: list[str] | None = None,
    ) -> list[SemanticMemory]:
        """
        조회 우선순위:
        1. agent_id 일치 + 높은 유사도
        2. agent_id = NULL (공유 지식) + 높은 유사도
        3. confidence 높은 순
        4. use_count 높은 순 (자주 쓰인 지식 우선)
        """
        embedding = await self.embed(query)

        results = await self.db.execute("""
            SELECT *, 1 - (embedding <=> :embedding) AS similarity
            FROM semantic_memories
            WHERE is_active = TRUE
              AND (agent_id = :agent_id OR agent_id IS NULL)
              AND (:memory_types IS NULL OR memory_type = ANY(:memory_types))
              AND 1 - (embedding <=> :embedding) >= :min_similarity
            ORDER BY
                CASE WHEN agent_id = :agent_id THEN 0 ELSE 1 END,
                similarity DESC,
                confidence DESC,
                use_count DESC
            LIMIT :max_results
        """, {
            "embedding": embedding,
            "agent_id": agent_id,
            "memory_types": memory_types,
            "min_similarity": self.RETRIEVAL_CONFIG["min_similarity"],
            "max_results": self.RETRIEVAL_CONFIG["max_results"],
        })

        # 조회 횟수 갱신
        await self.db.execute(
            "UPDATE semantic_memories SET use_count = use_count + 1 WHERE id = ANY(:ids)",
            {"ids": [r.id for r in results]}
        )

        return results
```

## 5.4 Semantic Memory 등록 규칙

```python
class SemanticMemoryWriter:

    async def register_knowledge(
        self,
        content: str,
        memory_type: str,
        agent_id: str | None,
        source_trace_id: UUID | None,
        approval_id: UUID,          # P1 승인 필수
        tags: list[str] = [],
        confidence: float = 0.9,
    ) -> SemanticMemory:
        """
        Semantic Memory 신규 등록 — Evol-A 또는 Human만 가능
        반드시 P1 ApprovalGate(G-09) 통과 후 호출
        """
        # 중복 확인 (유사도 95% 이상이면 기존 업데이트)
        existing = await self.retrieve(content, agent_id, min_similarity=0.95)
        if existing:
            return await self.update_version(existing[0].id, content, confidence)

        embedding = await self.embed(content)
        sanitized = sanitize_context(content)  # 민감정보 제거

        return await self.db.insert(SemanticMemory(
            agent_id=agent_id,
            memory_type=memory_type,
            content=sanitized,
            embedding=embedding,
            source_trace_id=source_trace_id,
            tags=tags,
            confidence=confidence,
        ))
```

# 6. Memory 생명주기 관리

## 6.1 전체 생명주기 흐름

```
[생성]
  Agent 실행 시작 → Session Memory 생성 (Redis)
  실행 완료 → Episodic Memory 저장 (PostgreSQL)
  패턴 학습 → Evol-A 분석 → Semantic Memory 등록 (승인 후)

[사용]
  Agent 실행 전 → Semantic Memory 조회 → 컨텍스트 주입
  실행 중 → Session Memory 상태 갱신
  실패 분석 → Episodic Memory 조회 → 원인 파악

[만료/삭제]
  Session Memory → TTL 자동 만료
  Episodic Memory → 보존 정책 기준 자동 삭제
  Semantic Memory → 신뢰도 하락 시 is_active=FALSE → 승인 후 삭제
```

## 6.2 개인정보 처리 방침

| 항목 | 기준 |
|------|------|
| PII 저장 | 원칙적 금지. 불가피한 경우 암호화 후 저장, 명시적 동의 필요 |
| PII 마스킹 | 저장 전 SENSITIVE_PATTERNS 필터 자동 적용 |
| 열람 권한 | 해당 도메인 Human 책임자 + PM 승인 (audit_log 필수) |
| 삭제 요청 | GDPR/개인정보보호법 기준, P1 ApprovalGate 후 처리 |
| 해외 이전 | 금지 (국내 서버 전용, 공공기관 적용 기준) |
| 보존 기간 | 목적 달성 즉시 최소화, 법정 보존 기간 준수 |

## 6.3 Memory 무결성 보장

```python
# Memory 접근 시 항상 아래 규칙 적용
class MemoryIntegrityChecker:

    def validate_access(self, agent_id: str, memory: Memory) -> bool:
        # 1. 해당 Agent의 접근 권한 확인
        if memory.agent_id and memory.agent_id != agent_id:
            raise MemoryAccessError(agent_id, memory.id)

        # 2. is_active 확인
        if not memory.is_active:
            raise MemoryDeprecatedError(memory.id)

        # 3. 보존 기간 만료 확인
        if memory.expires_at and memory.expires_at < datetime.now():
            raise MemoryExpiredError(memory.id)

        return True
```
