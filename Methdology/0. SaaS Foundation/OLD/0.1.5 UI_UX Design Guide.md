## 1. 설계 원칙

### 1.1 5대 원칙

| # | 원칙 | 내용 | 공공기관 적용 이유 |
|---|------|------|-----------------|
| **P1** | **투명성 우선** | AI가 무엇을 하고 있는지 항상 사용자에게 보여준다 | 감사 담당자(P3)가 AI 의사결정 근거를 즉시 확인 가능해야 함 |
| **P2** | **Human 통제 가시화** | 승인 대기·실행 중·완료 상태를 항상 명시한다 | AI 자율 실행에 대한 공공기관 신뢰 확보 |
| **P3** | **오류 예방 우선** | 비가역 액션 전 반드시 확인 단계를 제공한다 | 실수에 의한 공공 데이터 손상 방지 |
| **P4** | **정보 밀도 균형** | 전문가(P1·P2)와 비전문가(P5) 모두 이해 가능하도록 계층화 | Persona별 상이한 기술 수준 대응 |
| **P5** | **일관성 강제** | 모든 화면에서 동일한 컴포넌트·용어·레이아웃 사용 | 팀원마다 다른 구현 방지 (P2 Pain Point 해소) |

### 1.2 Persona별 UI 우선순위

| Persona | 핵심 Pain Point | UI 대응 방향 |
|---------|---------------|------------|
| P1 공공기관 AI 아키텍트 | AI 의사결정 기록 체계 없음 | Audit Trail UI, 실행 이력 타임라인 |
| P2 개발 조직 테크리드 | 표준 없어서 코드 스타일 불일치 | 컴포넌트 라이브러리 강제, Storybook |
| P3 보안/감사 담당자 | AI 근거 설명 불가, 로그 수집 4주 | 즉시 조회 가능한 감사 로그 대시보드 |
| P4 AI 운영 담당자 | Drift 감지 체계 없음 | 실시간 KPI 모니터링 위젯 |
| P5 규제 기관 담당자 | 기술 용어 중심으로 정책 검토 어려움 | 비기술 요약 뷰, 용어 툴팁 |

## 4. 공통 컴포넌트 명세

> 기반: **shadcn/ui** + **Tailwind CSS**  
> 위치: `frontend/src/components/ui/`  
> 규칙: shadcn/ui 기본 컴포넌트를 **반드시 래핑해서 사용** — 직접 수정 금지

### 4.1 Button

```tsx
// components/ui/Button.tsx
type ButtonVariant = 'primary' | 'secondary' | 'destructive' | 'ghost' | 'outline'
type ButtonSize    = 'sm' | 'md' | 'lg'

// ✅ 사용 예시
<Button variant="primary" size="md" onClick={handleSave}>저장</Button>
<Button variant="destructive" size="sm" isLoading={isDeleting}>삭제</Button>
<Button variant="ghost" leftIcon={<PlusIcon />}>추가</Button>

// 규칙
// - 비가역 액션(삭제·발송·배포)은 반드시 variant="destructive" 사용
// - 로딩 중에는 isLoading={true} — disabled 직접 제어 금지
// - 아이콘만 있는 버튼은 aria-label 필수
```

| variant | 색상 | 사용 시점 |
|---------|------|---------|
| `primary` | blue-600 | 주요 CTA (저장, 확인, 실행) |
| `secondary` | gray-100 | 보조 액션 (취소, 목록으로) |
| `destructive` | red-600 | 삭제·비가역 액션 |
| `ghost` | transparent | 테이블 내 액션, 아이콘 버튼 |
| `outline` | border-only | 필터, 태그 선택 |

### 4.2 Form / Input

```tsx
// react-hook-form + zod 조합 필수
import { useForm } from 'react-hook-form'

```

#### 입력 상태 색상

| 상태 | 테두리 | 배경 |
|------|--------|------|
| 기본 | `border-gray-300` | `bg-white` |

#### 테이블 표준 규칙

| size | 너비 | 사용 시점 |
|------|------|---------|

| `md` | 560px | 단순 폼 입력 |

| `full` | 100vw | 코드 리뷰, 전체 화면 필요 시 |

### 4.5 Toast / Alert

```tsx
// 4가지 유형
toast.success('저장되었습니다')

// 위치: 우측 하단 고정
// 자동 닫힘: success/info 3초, warning 5초, error 수동 닫기
// 스택: 최대 3개 동시 표시
```

Workflow의 Gate(G-01~G-09) 상태를 페이지 상단 또는 사이드패널에 표시한다.


## 7. 접근성 기준

### 7.1 필수 적용 항목

```tsx
// 다크모드 토글 컴포넌트
// components/ui/ThemeToggle.tsx
import { useTheme } from 'next-themes'

export function ThemeToggle() {
  const { theme, setTheme } = useTheme()
  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}
      aria-label="테마 전환" className="p-2 rounded-lg hover:bg-gray-100 dark:hover:bg-gray-800">
      {theme === 'dark' ? <SunIcon /> : <MoonIcon />}
    </button>
  )
}
```
